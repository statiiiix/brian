# Brian SEO and AI Visibility Audit

Date: 2026-07-11

## Executive summary

Brian has clear positioning, a single descriptive H1, a logical heading hierarchy, substantive product explanations, and an FAQ that answers real evaluation questions. The largest weaknesses were technical discovery and entity clarity rather than page structure.

The initial implementation had no canonical URL, sitemap, social metadata, structured data, AI-readable product file, or machine-readable pricing. The authenticated application and login routes also inherited the homepage's indexable metadata. These issues are addressed by the changes accompanying this report.

The remaining strategic constraint is that the public site is a client-rendered Create React App. Major search engines can render it, but server-rendered or pre-rendered homepage HTML would make discovery more reliable for smaller crawlers and AI agents and would reduce time to meaningful content.

## Findings and changes

### 1. Search metadata and entity signals

- Impact: High
- Evidence: The HTML shell initially had only a generic title and description.
- Change: Added a query-aligned title and description, canonical URL, full robots directives, Open Graph metadata, Twitter card metadata, and Organization, WebSite, SoftwareApplication, and FAQPage JSON-LD.
- Validation note: FAQ structured data mirrors visible page content. The production build replaces the site URL placeholder with the configured production origin.

### 2. Sitemap and canonical origin

- Impact: High
- Evidence: No XML sitemap existed and `robots.txt` did not advertise one.
- Change: The production build now emits `sitemap.xml`, adds its absolute URL to `robots.txt`, and replaces canonical/schema/social URL placeholders.
- Deployment requirement: Set `REACT_APP_SITE_URL` to the canonical production origin. Vercel production URL variables are used automatically as a fallback.

### 3. Private route indexation

- Impact: Medium
- Evidence: `/login` and `/app/*` use the same SPA shell as the public homepage.
- Change: Route-aware metadata now sets `noindex,nofollow` and non-marketing titles outside the homepage.

### 4. AI crawler access and extractability

- Impact: Medium
- Evidence: Crawling was generally allowed, but there was no explicit policy or AI-readable site summary.
- Change: Added explicit allow rules for major search-and-citation crawlers, `llms.txt`, and a direct, self-contained product definition in the hero copy.

### 5. Machine-readable commercial information

- Impact: Medium
- Evidence: Pricing existed only inside the rendered React page and numeric pricing has not launched.
- Change: Added `pricing.md` with honest availability, inclusions, compatibility, and contact details. It explicitly says public numeric pricing is not yet available rather than inventing a price.

### 6. Web app identity

- Impact: Low
- Evidence: The web manifest still used Create React App sample names and colors.
- Change: Updated the app name, short name, start URL, and brand colors.

## Prioritized next actions

1. Set and verify the canonical production domain through `REACT_APP_SITE_URL`, deploy, and submit `/sitemap.xml` in Google Search Console and Bing Webmaster Tools.
2. Pre-render or server-render the public homepage. Keep authenticated product routes client-rendered and noindexed.
3. Create a 1200×630 social preview image that communicates “governed AI agent skills”; the current metadata uses the square product logo as a safe fallback.
4. Add a public proof or methodology section using Brian's reproducible retrieval benchmark. Original data is substantially more citation-worthy than generic marketing claims, but the public wording should remain synchronized with the latest benchmark.
5. Publish focused, indexable pages for category and comparison intent, starting with “company brain for AI agents,” “AI agent governance,” “Brian vs agent memory,” and “MCP skills for AI agents.” Avoid creating thin keyword variants.
6. Establish a monthly AI visibility baseline across the priority queries in ChatGPT search, Perplexity, Google AI Overviews, Gemini, and Copilot. Record cited domains and cited pages, not only brand mentions.
7. Add verified founder/author identity, company social profiles, legal pages, and a durable company contact page as they become available. These improve trust and entity reconciliation.

## Measurement plan

Track indexed pages, non-brand impressions, query clusters, Core Web Vitals, and sitemap status in Search Console. Separately track AI citation rate, cited Brian URLs, competitor citation share, and citation sentiment for a fixed monthly query set. Google does not provide a dedicated AI Overview report, so AI visibility needs a separate manual or third-party tracker.
